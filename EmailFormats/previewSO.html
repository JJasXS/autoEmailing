previewSO — attachment format (Excel + PDF)

The worker sends the usual HTML intro, plus two attachments when App:DailyAttachmentReport:Enabled is true and Sql is set: Outstanding_yyyy-MM-dd.xlsx and Outstanding_yyyy-MM-dd.pdf. The same query runs once per batch; every recipient gets the same files (filenames include the schedule date only).

Local HTTP preview (port 2086): from the project or publish folder run dotnet run -- --preview-so — then open http://localhost:2086/ (default document is so-transfer-preview.html from the listener). Override port: set PREVIEW_SO_PORT=2086 (or another port) then run the same command.
If you use the EC2 public DNS or IP in the URL and see 400 Invalid Hostname, set PREVIEW_SO_ANY_HOST=1 in .env and on Windows run once (elevated): netsh http add urlacl url=http://+:2086/ user=Everyone (match the port).
SO transfer report (Excel/PDF): /so-transfer (same server).

Install (one time, on the dev/build machine): already referenced in the project — ClosedXML (Excel) and QuestPDF (PDF). Restore with dotnet restore. QuestPDF uses the Community license flag in Program.cs; review QuestPDF licensing for your organization.

1) Required SQL column aliases

Your Firebird SELECT must expose these four columns (aliases are matched case-insensitively):

Alias Meaning Suggested type
DOCUMENTInvoice / document referenceVARCHAR
DOC_DATEDocument or due dateDATE or TIMESTAMP
AGE_DAYSAge in whole daysINTEGER
AMOUNTOutstanding amountNUMERIC

Replace the sample Sql in appsettings.json with your real outstanding query; keep the four aliases on the outermost select.

2) Sample Firebird SQL (demo rows — works on any database)

SELECT CAST('INV-DEMO-1' AS VARCHAR(120)) AS DOCUMENT,
       CAST(DATE '2025-01-10' AS DATE) AS DOC_DATE,
       CAST(12 AS INTEGER) AS AGE_DAYS,
       CAST(100.00 AS NUMERIC(18,4)) AS AMOUNT
FROM RDB$DATABASE
UNION ALL
SELECT CAST('INV-DEMO-2' AS VARCHAR(120)) AS DOCUMENT,
       CAST(DATE '2025-02-01' AS DATE) AS DOC_DATE,
       CAST(5 AS INTEGER) AS AGE_DAYS,
       CAST(250.50 AS NUMERIC(18,4)) AS AMOUNT
FROM RDB$DATABASE

4) Turn attachments on

In appsettings.json or environment overrides, set:

App__DailyAttachmentReport__Enabled=true

Optional: App__DailyAttachmentReport__ReportTitle, App__DailyAttachmentReport__CompanyName, App__DailyAttachmentReport__CurrencyLabel, App__DailyAttachmentReport__Sql (multi-line SQL is easiest in appsettings; escape quotes in JSON).
